# 🎧 Teams 会议实时翻译 · Live Meeting Translator

开会时别人讲英语，侧边栏实时显示中文字幕（大字中文 + 小字英文原文，**话说到一半中文就开始出现**）。

这是一个标准的 **Microsoft Teams 会议侧边栏应用**（meeting side panel app），也可以不装进 Teams、直接当网页用。纯前端、无服务器：页面托管在 GitHub Pages，语音翻译从浏览器直接调用 Azure Speech（Teams 官方字幕用的同一个引擎），所有 Key 只保存在本机浏览器。

## 快速开始（三步）

1. **开启 GitHub Pages**：本仓库 Settings → Pages → Source 选「Deploy from a branch」→ Branch 选 `main` → Save。
   几分钟后确认能打开 `https://alan765477.github.io/teams-live-translator/`
2. **填 Key**：打开上面的网址，点右上 ⚙️，填入 Azure Speech Key + 区域（portal.azure.com 免费创建「语音服务」资源，**免费额度每月 5 小时音频**）
3. **开会时用**（二选一）：
   - **免安装（推荐先试这个）**：开会时旁边开个浏览器标签打开本页面 → 选「🔊 会议声音」→ 开始 → 共享 Teams 所在标签页/屏幕并**勾选「共享音频」**
   - **装进 Teams**：见下

> 如果你的 GitHub 用户名或仓库名不是 `alan765477 / teams-live-translator`，把 `manifest.json` 里的所有网址和 `validDomains` 改成你的，再运行 `sh package.sh` 重新打包。

## 安装进 Teams（工作/学校账号）

| 你的账号类型 | 能否安装 |
|---|---|
| 公司/学校 Teams（工作账号） | ✅ 可以，前提是 IT 管理员允许「上传自定义应用」（多数公司默认允许；不行就把 `teams-app.zip` 发给 IT 帮你装） |
| 个人免费版 Teams | ❌ 不支持自定义会议应用——用上面的免安装用法，效果一样 |

步骤：

1. 下载仓库里现成的 **`teams-app.zip`**（改过 manifest 就运行 `sh package.sh` 重新生成）
2. Teams 左侧「应用」→「管理你的应用」→「上传应用」→「上传自定义应用」→ 选 `teams-app.zip`
3. 进入（或预约）会议 → 顶部「+ 应用」→ 选「实时翻译」→ 出现在会议右侧边栏
4. 第一次用点右上 ⚙️ 填 Azure Key

## 声音从哪来（重要）

插件本质是网页，它有两种方式"听到"会议：

| 模式 | 原理 | 适用场景 |
|---|---|---|
| 🔊 **会议声音（推荐）** | 浏览器屏幕共享的"共享音频"功能，直接捕获会议声音流 | **在浏览器里开会**（teams.microsoft.com，Edge/Chrome），戴不戴耳机都行，听得最清楚 |
| 🎤 **麦克风** | 用麦克风收音 | 现场会议、或电脑外放开会；戴耳机时收不到对方声音 |

> Teams **桌面客户端**的侧边栏不允许网页捕获系统声音，推荐两种姿势：① 用浏览器版 Teams 开会 + 侧边栏插件；② 桌面客户端开会 + 旁边浏览器标签开本页面（免安装用法）。

## 翻译引擎

- **Azure Speech（强烈推荐）**：一条流式接口同时完成"识别英语 + 翻译中文"，延迟最低，也是唯一支持"会议声音"模式的引擎。免费层每月 5 小时，个人开会足够。
- **AI 翻译（备用）**：没有 Azure 时，麦克风模式可用浏览器语音识别 + Claude/DeepSeek 逐句翻译，每句话说完约 1 秒出中文。

## 功能一览

- **自选语言**：「对方说 / 我说」两个选择器，支持中文、English、日本語、한국어、Español、Français、Deutsch、Русский，任意组合；识别语言、翻译方向、朗读音色全部自动匹配
- **三种模式**（顶部分段控件）：
  - **听对方**：会议声音或麦克风 → 出你的语言的字幕
  - **我来说**：你说自己的语言 → 出对方语言的字幕
  - **双向对话**：同时开两路——会议声音（标「对方」蓝色）+ 你的麦克风（标「我」绿色），全程自动区分发言方并双向互译。需 Azure Key
- **当前句居中大字**显示，历史记录在上方淡化滚动
- **⏺ 录音**：把你的麦克风和会议声音混成一轨录制，停止后自动下载音频文件
- **🎥 录屏**：会议画面 + 会议声音 + 你的麦克风录成一个视频文件，开完会回看复盘。支持直接边录边写入磁盘（长会不占内存）；约每小时 0.5~1GB。⚠️ 录音/录屏请先征得参会人同意
- **📝 纪要**：散会点一下，AI 把全部字幕整理成「主题/要点/决定/待办」，可复制、可下载 Markdown（需 AI Key）；纪要面板里还能一键下载**全部字幕原文+译文**（无需任何 Key）
- **🔈 朗读**：见下节

> 关于"显示谁在说话"：网页能拿到的会议声音是混合后的一路，无法拆分对方多人（能拆分的产品都是用机器人入会拿每人独立音频流）。所以标注做到「我 / 对方」两方；如需"对方1/对方2"级别的区分，需要升级到 Azure ConversationTranscriber 声纹分离，可以以后加。

## 语音朗读 / 同声传译（🔈 朗读翻译）

开启「🔈 朗读翻译」后，每句翻译完成会用语音读出来（配 Azure 时是真人级神经音色：中→英读英语、英→中读中文）。三种玩法：

1. **自己听**（默认走扬声器/耳机）：「英语→中文」+ 朗读 = 同传耳机效果，闭着眼也能"听懂"英语会议
2. **外放给对方听**：「中文→英语」+ 朗读，现场交流时手机/电脑外放
3. **直接送进 Teams 会议（真·同声传译）**：需要装一个免费的虚拟声卡：
   - Windows：VB-Audio Virtual Cable（vb-audio.com/Cable）
   - Mac：BlackHole（existential.audio/blackhole）

   步骤：装好后 → 翻译器 ⚙️「朗读输出设备」选 **CABLE Input** → Teams 设置 → 设备 → 麦克风选 **CABLE Output** → 界面选「中文→英语」+ 开启朗读。你说中文，会议里对方直接听到英语语音（你的中文原声不会传进会议）。

   > 注意：指定朗读输出设备需要 Azure Key（浏览器自带语音只能走默认扬声器）；虚拟声卡装完要重启浏览器才会出现在设备列表里。

## 业界是怎么做这类产品的？

1. **不做插件，做进产品**（微软自己）：Teams Premium 内置实时翻译字幕——有 Premium 的话开会点「更多 → 语言和语音」直接开，什么都不用装。
2. **实时媒体机器人**（Otter.ai、讯飞同传等专业产品）：Graph 通信 API 写机器人以参会者身份入会，服务器拿到每个发言人的独立原始音频流（所以能标注谁在说），服务端识别+翻译再推回侧边栏。效果最好，但要 Azure Bot、C#/Windows 服务器、管理员批准，复杂度百倍。
3. **Graph 实时转录 API**：订阅 Teams 自己的会议转录流，只做翻译一步，仍需服务器和授权。
4. **客户端捕获声音**（本项目）：直接捕获"你听到的声音"。零服务器、零审批、零成本，缺点是无法区分发言人。个人自用体验非常接近方案 2（引擎相同，延迟差一秒以内）。
