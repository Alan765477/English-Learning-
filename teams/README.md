# Teams 会议实时翻译插件

开会时侧边栏实时显示翻译字幕，**英↔中双向**：别人讲英语出中文字幕，讲中文出英文字幕，支持自动识别语言方向。这是一个标准的 **Microsoft Teams 会议侧边栏应用**（meeting side panel app），也可以不装进 Teams、直接当网页用。

![原理](#工作原理) 纯前端、无服务器：页面部署在 GitHub Pages 上，语音翻译直接从浏览器调用 Azure Speech（微软 Teams 官方字幕用的同一个引擎），Key 只存在本机。

---

## 一、可以直接安装到 Teams 里吗？

**可以，但不是从应用商店装，而是「上传自定义应用」（sideload）**，条件如下：

| 你的账号类型 | 能否安装 |
|---|---|
| 公司/学校的 Teams（工作账号） | ✅ 可以，前提是 IT 管理员开启了「允许上传自定义应用」（多数公司默认允许开发者上传；不行就把 `teams-app.zip` 发给 IT 帮你装） |
| 个人免费版 Teams | ❌ 不支持自定义会议应用 —— 用下面的 **免安装用法** 即可，效果一样 |

**免安装用法（任何账号都行，也最稳定）**：开会时在旁边开一个浏览器标签，打开
`https://alan765477.github.io/English-Learning-/teams.html`，选「🔊 会议声音」→ 开始 → 共享 Teams 所在的标签页/屏幕并勾选「共享音频」。中文字幕实时滚出。

## 二、安装步骤（工作/学校账号）

1. **部署网页**：本仓库开启 GitHub Pages（Settings → Pages → Deploy from branch → main）。部署后确认能打开 `https://alan765477.github.io/English-Learning-/teams.html`
   > 如果你的用户名/仓库名不同，改 `manifest.json` 里的所有网址和 `validDomains`，然后重新打包
2. **拿到安装包**：直接用仓库根目录现成的 `teams-app.zip`，或自己运行 `sh teams/package.sh` 重新生成
3. **上传到 Teams**：Teams 左侧「应用」→「管理你的应用」→「上传应用」→「上传自定义应用」→ 选 `teams-app.zip`
4. **加进会议**：进入（或预约）一个会议 → 顶部「+ 应用」→ 选「实时翻译」→ 它会出现在会议右侧边栏
5. 第一次用点右上 ⚙️ 填 **Azure Speech Key**（见下）

## 三、声音从哪来（重要）

插件在侧边栏里是一个网页，它有两种方式"听到"会议：

| 模式 | 原理 | 适用场景 |
|---|---|---|
| 🔊 **会议声音（推荐）** | 浏览器屏幕共享的"共享音频"功能，直接捕获会议的声音流 | **在浏览器里开会**（teams.microsoft.com，Edge/Chrome），戴不戴耳机都行，听得最清楚 |
| 🎤 **麦克风** | 用麦克风收音 | 现场会议、或电脑外放开会时；戴耳机时收不到对方声音 |

> Teams **桌面客户端**的侧边栏不允许网页捕获系统声音，所以推荐两种姿势：① 用浏览器版 Teams 开会 + 侧边栏插件；② 桌面客户端开会 + 旁边浏览器标签开本页面（免安装用法）。

## 四、翻译引擎

- **Azure Speech（强烈推荐）**：一条流式接口同时完成"识别英语 + 翻译中文"，**对方话说到一半中文就开始出来**。portal.azure.com 创建免费「语音服务」资源即可，**免费层每月 5 小时音频**，个人开会足够。也是唯一支持"会议声音"模式的引擎。
- **AI 翻译（备用）**：没有 Azure 时，麦克风模式可用浏览器语音识别 + Claude/DeepSeek 逐句翻译（和主 App 的实时翻译共用 Key），每句话说完约 1 秒后出中文。

所有 Key 只保存在本机浏览器 localStorage，不经过任何服务器。

## 五、Apple / 顶级公司会怎么做？

做一个"能听到会议里所有人说话"的产品级翻译插件，业界有四条路，从轻到重：

1. **不做插件，做进产品**（微软自己的做法）：Teams Premium 订阅内置「实时翻译字幕」，因为微软坐在音频管道上，根本不需要"捕获"声音。**如果你公司有 Teams Premium，开会点 更多 → 语言和语音 → 开启实时字幕并选翻译语言，什么都不用装。**
2. **实时媒体机器人（real-time media bot）**——Otter.ai、讯飞同传、Interprefy 这类专业产品的做法：用 Microsoft Graph 通信 API 写一个机器人，以"参会者"身份加入会议，在服务器上拿到**每个发言人独立的原始音频流**（所以能标注谁在说话），服务端做识别+翻译，再推回侧边栏应用显示。效果最好，但需要 Azure Bot 注册、C#/Windows 服务器、公司管理员批准 API 权限，成本和复杂度是纯前端方案的百倍。
3. **Graph 实时转录 API**：较新的折中——订阅 Teams 自己生成的会议转录流，只做"翻译"这一步。仍需服务器和管理员授权。
4. **客户端捕获声音**（本插件的做法，也是 Tactiq、部分浏览器翻译插件的做法）：不碰 Teams 的音频管道，直接在用户电脑上捕获"你听到的声音"。零服务器、零审批、零成本，缺点是无法区分发言人、依赖浏览器能力。

对个人自用，方案 4 的体验已经非常接近方案 2（引擎相同，都是 Azure Speech，延迟差别在一秒以内），所以本插件选了它。
